# For Teams

*En español: [para-equipos/README.md](../para-equipos/README.md)*

The rest of this repo is written for individuals. One person, one vault, one brain that compounds.

This folder is for teams. Co-founders, operating partners, Chiefs of Staff, small-company execs who read the root README and wondered what the system looks like when more than one person uses it.

## Quick orientation

If you have not read the root [README](../README.md) yet, start there. This folder assumes you already understand what the personal version does. It explains what changes when you try to run it across a team, and how to install the team version without figuring out every tradeoff yourself.

## What is in here

- **[Why Teams Are Different](why-teams-are-different.md)** lists the four problems the personal version never had to solve, in the order they usually break things.
- **[Team Workflows](team-workflows.md)** walks through four workflows a team vault runs that a personal vault cannot. Concrete, specific, the kind you can map directly to things your team is already doing badly.
- **[Working With Me](working-with-me.md)** is the menu if you want the team version installed and customized instead of building it yourself. Three packages, founding rate, risk reversal, and the one-link path to a free 20-minute diagnostic.

## How to use this folder

1. Skim **Why Teams Are Different**. If none of the four problems sound like your team, you probably do not need the team version. Keep exploring the personal version in the rest of the repo.
2. If one or two of those friction points hit, read **Team Workflows**. See if any of the four workflows match what your team is already trying to do and failing at.
3. If you want the team version built for you instead of figuring it out yourself, open **Working With Me**.
